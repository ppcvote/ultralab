Authentication、Firestore、Hosting。

寫 prompt 跟寫程式其實很像。

你以為你講清楚了，但 AI 理解的完全不是那回事。

所以 prompt 工程的核心能力不是「會下指令」，是「會想清楚」。

「Playwright 不是用來寫 E2E 測試的嗎？」

是。但它也能截圖、錄影、操作瀏覽器。
我們拿它來把 HTML 模板變成影片素材。

工具的價值不在於它被設計來做什麼，在於你拿它做什麼。
---
為什麼我們堅持用 Tailwind CSS：

1. 不用想 class 命名
2. 響應式一行搞定
3. 新人看 code 直接看得懂樣式
4. 產出的 CSS 比手寫的還小

少打幾個字，多睡幾分鐘。
---
SaaS 建置的第一步不是寫 code。
是畫出「用戶從註冊到付費」的完整流程圖。

如果這張圖畫不出來，寫 code 也只是在浪費時間。
---
我們內部開會的規矩：
超過 15 分鐘的會議要有議程。
沒有結論的會議等於沒開。
能用訊息解決的不要開會。

省下來的時間拿去寫 code。
---
FFmpeg 指令大概是世界上最難讀的東西之一。

但學會之後你會發現：
影片剪接、轉檔、合成、加字幕，全部一行指令搞定。

學習曲線陡，但回報率高到離譜。

自動化不是一次到位的。

先把最痛的那個環節自動化。
跑穩了再處理下一個。
循環三到五次，整條流程就通了。

急著一次做完的人，通常一個環節都做不完。

我們對「MVP」的定義：

能看 — UI 不丟人

三個條件都滿足，就該上線了。
其他的，看數據再說。

API 串接最常見的坑：

1. Rate limit 沒處理 → 直接被 ban
2. 錯誤回傳沒 parse → 用戶看到 500
3. Token 過期沒刷新 → 半夜服務中斷

這三個坑，我們每個都踩過。
所以現在我們的 boilerplate 裡全都有。
