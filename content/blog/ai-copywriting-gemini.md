## 為什麼用 AI 生成社群文案？

每天手動寫 5-10 篇社群文案，是一件極度消耗心力的事。尤其當你要同時經營多個帳號、多個主題時，「今天要寫什麼」的靈感枯竭感會讓你痛不欲生。

AI 文案生成不是要取代你的創意，而是要**把 0→60 分的苦力活交給機器**，你只需要從 60 分修到 90 分。

我們在 Ultra Lab 的實際數據：
- 6 個 Threads 帳號，每天 35+ 篇貼文
- 全部由 Gemini API 自動生成
- 人工介入時間：**每週 15 分鐘**（抽查品質、調整方向）

## Prompt 工程：決定文案品質的 80%

AI 的輸出品質，80% 取決於你怎麼問它。以下是我們在實戰中驗證過的 Prompt 設計框架：

### 框架一：角色 + 任務 + 格式 + 限制

```
你是一個台灣的 {領域} 專家，用繁體中文寫作。
語調：{輕鬆幽默 / 專業但親切 / 犀利直白}

請寫一篇 Threads 貼文，主題是「{主題}」。

要求：
- 字數 100-200 字
- 開頭用一句引人注目的 hook
- 結尾加一個引發互動的問題
- 不要用 emoji（或：適當使用 1-2 個 emoji）
- 不要用「在這個...的時代」之類的 AI 感開頭
```

### 框架二：給範例（Few-shot）

最有效的方式是直接給 AI 看你之前寫得好的文案，讓它學習你的風格：

```
以下是我之前寫的 3 篇表現最好的 Threads 貼文：

範例 1：{高互動貼文}
範例 2：{高互動貼文}
範例 3：{高互動貼文}

請用相同的風格和語調，寫一篇關於「{新主題}」的貼文。
```

### 框架三：批次生成

一次生成多篇，再挑最好的：

```
請用以下主題，各生成 3 個版本的 Threads 貼文。
每個版本用不同的切角（觀點、比喻、數據）。

主題 1：{主題}
主題 2：{主題}
主題 3：{主題}
```

## API 串接的技術細節

### 基本呼叫

用 Gemini API 生成一篇文案的基本流程：

1. 準備 Prompt 模板（含品牌語調、格式要求）
2. 帶入當天的主題/關鍵字
3. 呼叫 API 取得回應
4. 後處理（去除多餘的格式符號、統一標點符號）
5. 存入資料庫等待發布

### 批次處理的注意事項

- **Rate Limiting**：Gemini 免費版有每分鐘請求上限，需要做好排隊機制
- **錯誤處理**：API 偶爾會回傳空白或不符格式的結果，需要重試邏輯
- **內容過濾**：AI 偶爾會產出不適當的內容，建議加上關鍵字過濾

### 溫度參數（Temperature）

- **Temperature 0.3-0.5**：適合事實性內容（數據分析、教學文）
- **Temperature 0.7-0.9**：適合創意性內容（觀點文、故事文）
- **Temperature 1.0+**：太發散，不建議用於品牌文案

## 品質控制：不能全部放手

即使 AI 生成品質很高，你仍然需要做以下品質控制：

### 每週抽查

從本週自動生成的文案中，隨機抽 5-10 篇檢查：
- 事實是否正確？（AI 偶爾會編造數據）
- 語調是否一致？
- 有沒有重複的內容結構？

### 數據回饋

追蹤每篇文案的互動數據（按讚、留言、分享、追蹤轉換），找出：
- 哪些主題最受歡迎？
- 哪種開頭 hook 效果最好？
- 哪個發布時段互動最高？

把這些洞察回饋到 Prompt 模板中，形成**持續優化的飛輪**。

### 定期更新 Prompt

不要一套 Prompt 用到天荒地老。每 2-4 週根據數據調整：
- 加入新的高互動範例
- 調整語調方向
- 新增或淘汰主題
